2. Определить, в каких файлах не содержится подстрока (определённая)

```
grep -rlvw ./ -e aaa
```

`grep` - даёт возможность сортировать и фильтровать текст на основе данных правил

`l` - вывести только имена файлов, в которых найден шаблон

`v` - инвертировать поиск, выдавать все строки кроме тех, что содержат шаблон. Так как мы ищем те файлы, где не содержится упомянутая подстрока

`w` - искать шаблон как слово, окружённое пробелами. `aaa`, как пример

`e` - использует регулярные выражения при поиске

`r` - рекурсивный поиск (то есть по всем каталогам текущего каталога). В задании не сказано, с `r` или без

`aaa` - наша подстрока, которой не должно содержаться в найденных файлах

3. В каталоге /home/user/workspace создать каталог test, где будет файл test.txt

```
cd /home/user/workspace
mkdir ./test
cd ./test
touch test.txt
```

`mkdir` - создаёт директорию

`touch` - создаёт файл

4. Сделать архив, содержащий каталог test

```
cd /home/user/workspace
tar -cf archive.tar ./test
```

`tar` - архивирует файлы

`c` - параметр добавления файлов в архивирует

`f` - выводит результат в file (без него не работает)

5. Найти все файлы текущего каталога, где встречается подстрока `aaa` и подстрока `bbb` одновременно

```
grep -lwr ./ -e aaa | grep -lwr ./ -e bbb
```

Я нахожу, что добавление описания каждого параметра занимает много времени. Из ЛР01 ясно, что делает каждый из параметров `grep` здесь

`|` - *перевпихивает* вывод команды в аргумент новой

6. Определить количество только каталогов в текущем каталоге

```
find . maxdepth 1 -type d | wc
```

можно также через `grep`

`^d` - здесь символизирует, что файл заканчивается на d, то есть директории

10. Что будет, если произойдёт следующее ?

```
git checkout master
git merge patch1
```

`git merge` сливает ветку `patch1` с текущей `master`

11. Что будет, если произойдёт следующее ?

```
git checkout patch1
git rebase master
```

`rebase` - это аналог `merge`. Логика та же. Что-то сливается с текущей веткой. Здесь ветка `master` вливается в `patch1`
